# OpenClaw 문서 한 번에 이해하기

## 🧠 핵심 요약

**OpenClaw는 여러 메신저와 AI 코딩 에이전트를 연결하는 자체 호스팅 게이트웨이입니다.**

- Discord, Telegram, WhatsApp, Slack, iMessage, Matrix 등 다양한 채널을 한 곳에서 다룹니다.
- 하나의 Gateway 프로세스로 세션, 라우팅, 채널 연결을 통합합니다.
- 모바일 노드, 웹 대시보드, CLI, 플러그인까지 함께 제공합니다.
- 데이터와 실행 환경을 직접 통제할 수 있어, 호스팅 서비스 의존도를 줄입니다.

## 🔍 OpenClaw가 뭐예요?

**OpenClaw는 내 기기나 서버에서 직접 돌리는 AI 메시징 허브입니다.**

- 사용자는 채팅 앱에서 메시지만 보내면 됩니다.
- Gateway가 이를 받아 AI 에이전트와 연결합니다.
- 코드 작업, 세션 관리, 멀티 채널 응답을 한 흐름으로 처리합니다.

## ⚙️ 어떻게 동작하나요?

**구조는 단순합니다. 채팅 앱이 들어오고, Gateway가 받아서 에이전트와 도구로 분배합니다.**

- 입력: 채팅 앱 + 플러그인 채널
- 중심: Gateway
- 출력: Pi 에이전트, CLI, Web Control UI, macOS 앱, iOS/Android 노드

**핵심은 Gateway가 모든 세션과 라우팅의 단일 기준점이라는 점입니다.**

## 🚀 주요 기능

**OpenClaw의 강점은 “한 번 연결하면 여러 방식으로 확장된다”는 점입니다.**

- 멀티 채널 게이트웨이, 하나의 프로세스로 여러 메신저 처리
- 플러그인 채널, Matrix나 Nostr 같은 추가 채널 확장
- 멀티 에이전트 라우팅, 작업자별 분리된 세션 운영
- 미디어 지원, 이미지·오디오·문서 송수신
- Web Control UI, 브라우저 기반 설정과 세션 관리
- 모바일 노드, 카메라·음성·Canvas 워크플로 지원

## 🛠️ 시작 방법

**설치는 비교적 짧고, 기본 흐름도 명확합니다.**

- `npm install -g openclaw@latest`
- `openclaw onboard --install-daemon`
- `openclaw dashboard`

**즉시 채팅을 시작하려면 Telegram 연결이 가장 빠른 편입니다.**

## 🧩 설정 포인트

**기본값만으로도 동작하지만, 보안과 제어를 위해 설정을 다듬는 편이 좋습니다.**

- 설정 파일 위치는 `~/.openclaw/openclaw.json`
- 허용 발신자 제한은 `channels.whatsapp.allowFrom` 같은 값으로 관리합니다.
- 그룹에서는 멘션 필수 규칙을 둘 수 있습니다.
- [추정] 보안 민감 환경에서는 allowlist와 멘션 규칙을 먼저 적용하는 편이 안전합니다.

## 🌐 대시보드와 원격 접근

**로컬 웹 대시보드가 기본 진입점입니다.**

- 기본 주소는 `http://127.0.0.1:18789/`
- 원격은 Web surfaces나 Tailscale로 확장합니다.
- 브라우저에서 채팅, 설정, 세션, 노드를 같이 관리할 수 있습니다.

## 🔐 왜 주목할 만한가요?

**OpenClaw는 “메신저 + 에이전트 + 자동화”를 한 덩어리로 묶는 점이 강합니다.**

- 자체 호스팅이라 데이터 통제가 쉽습니다.
- 채널이 많아도 Gateway 하나로 묶을 수 있습니다.
- 세션, 메모리, 작업 흐름이 에이전트 친화적으로 설계되어 있습니다.
- 플러그인 구조라 기능 확장이 유리합니다.

## 🧭 정리

**OpenClaw는 메시징을 에이전트 작업 흐름으로 바꾸는 인프라입니다.**

- 개인용 AI 비서
- 멀티 채널 메시징 허브
- 자동화와 백그라운드 작업 관리
- 모바일과 웹까지 이어지는 통합 운영

**한 줄로 말하면, OpenClaw는 “내 채팅 앱을 AI 작업 공간으로 바꾸는 게이트웨이”입니다.**
