## 📋 핵심 요약

### 주요 변경사항

- **이미지 생성 강화**: Codex OAuth 및 OpenRouter를 통한 이미지 생성/편집 지원, `OPENAI_API_KEY` 없이 `openai/gpt-image-2` 사용 가능
- **에이전트 개선**: `sessions_spawn`에 선택적 포크된 컨텍스트 추가, 이미지/비디오/음악/TTS 생성에 `timeoutMs` 지원
- **메모리 최적화**: 로컬 임베딩 `contextSize` 설정 가능 (기본값 4096), `openclaw memory` 명령어에서 로컬 임베딩 해결
- **Pi 업데이트**: Pi 패키지 0.70.0로 업데이트, `gpt-5.5` 카탈로그 메타데이터 사용

### 주요 버그 수정

- **WhatsApp/Slack**: 초기 설정 로딩 개선, 그룹 채팅에서 노이즈 감소
- **미디어 처리**: 텔레그램에서 마크다운 이미지 구문 파싱, 텍스트 전용 모델에서 이미지 첨부 유지
- **보안 강화**: Android/Teams/Discord/QQBot 플러그인 보안 개선, 인증 및 권한 검증 강화
- **메모리 복구**: QMD 컬렉션 재생성, `MEMORY.md` 기반 루트 메모리 정규화
- **플러그인**: Windows 플러그인 SDK 별칭 새로고침, 종속성 해결 개선

## 🎨 이미지 생성 강화

이번 업데이트의 하이라이트는 이미지 생성 기능의 대폭적인 강화입니다. 

**Codex OAuth 및 OpenRouter 통합**을 통해 이제 별도의 `OPENAI_API_KEY` 없이도 `openai/gpt-image-2` 모델을 사용할 수 있게 되었습니다. 이는 사용자 환경 설정을 간소화하고, 더 쉽게 AI 이미지 생성을 시작할 수 있게 해줍니다.

또한 이미지 편집 기능도 개선되어, 더 유연하고 강력한 이미지 생성 워크플로우를 구축할 수 있습니다.

## 🤖 에이전트 개선

에이전트 시스템도 상당한 개선이 이루어졌습니다:

- `sessions_spawn`에 **선택적 포크된 컨텍스트**가 추가되어, 에이전트 간 컨텍스트 공유가 더 유연해졌습니다
- 이미지, 비디오, 음악, TTS 생성 작업에 `timeoutMs` 파라미터를 지원하여, 긴 생성 작업의 타임아웃을 제어할 수 있게 되었습니다

## 🧠 메모리 최적화

메모리 시스템의 성능과 유연성이 개선되었습니다:

- 로컬 임베딩의 `contextSize`를 설정할 수 있게 되어, 기본값 4096 토큰에서 필요에 따라 조절 가능합니다
- `openclaw memory` 명령어에서 로컬 임베딩을 직접 해결할 수 있어, 메모리 검색이 더 빨라졌습니다
- QMD 컬렉션 재생성 및 `MEMORY.md` 기반 루트 메모리 정규화를 통해 메모리 무결성이 강화되었습니다

## 🔒 보안 강화

여러 플러그인에서 보안이 대폭 강화되었습니다:

- **Android**, **Teams**, **Discord**, **QQBot** 플러그인에서 인증 및 권한 검증이 개선되었습니다
- 이를 통해 사용자 데이터 보호와 시스템 보안이 더욱 강화되었습니다
