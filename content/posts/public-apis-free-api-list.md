# Public APIs: 무료 API를 찾을 때 가장 먼저 열어볼 저장소

새로운 앱이나 사이드 프로젝트를 만들 때 가장 먼저 막히는 지점이 있습니다. “쓸 만한 데이터는 어디서 가져오지?”라는 문제입니다. 날씨, 환율, 뉴스, 영화, 동물 이미지, 보안 정보, 테스트 데이터처럼 작은 기능 하나에도 외부 API가 필요할 때가 많습니다.

이럴 때 유용한 저장소가 GitHub의 **public-apis/public-apis**입니다. 이름 그대로 무료로 사용할 수 있는 공개 API를 분야별로 모아둔 큐레이션 목록입니다.

![public-apis repository cover](/images/public-apis-cover.png)

## 40만 스타가 넘는 API 보물창고

public-apis는 GitHub에서 40만 개가 넘는 star를 받은 대표적인 개발자 리소스입니다. 단순한 링크 모음처럼 보이지만, 실제로는 프로젝트 아이디어를 빠르게 검증할 때 꽤 강력합니다.

목록은 동물, 애니메이션, 보안, 디자인, 인증, 블록체인, 책, 비즈니스, 캘린더, 클라우드 스토리지, 암호화폐, 환율, 개발, 문서, 이메일, 금융, 음식, 게임, 지도, 정부, 건강, 머신러닝, 음악, 뉴스, 오픈데이터, 프로그래밍, 과학, 스포츠, 교통, 날씨 등 다양한 카테고리로 나뉘어 있습니다.

각 API는 보통 설명, 인증 방식, HTTPS 지원 여부, CORS 지원 여부를 함께 보여줍니다. 그래서 브라우저에서 바로 붙일 수 있는지, API key가 필요한지, 서버 프록시가 필요한지 빠르게 판단할 수 있습니다.

## 개발자가 쓰기 좋은 이유

이 저장소의 장점은 “검색 비용”을 줄여준다는 점입니다. 구글에서 API를 하나씩 찾다 보면 오래된 문서, 유료 전환된 서비스, 인증 방식이 복잡한 서비스가 섞입니다. public-apis는 커뮤니티가 관리하는 목록이라, 최소한 출발점으로 삼기 좋습니다.

예를 들어 간단한 대시보드를 만들고 싶다면 환율·날씨·뉴스 API를 빠르게 찾을 수 있습니다. 포트폴리오 프로젝트를 만든다면 영화, 음악, 게임, 스포츠 API를 붙여볼 수 있습니다. 테스트용 데이터가 필요하면 임시 사용자, 랜덤 이미지, 문장 분석 API를 찾아볼 수도 있습니다.

특히 초보 개발자에게는 “API를 써보는 감각”을 익히기 좋은 자료입니다. 인증 없는 API부터 시작해서, API key 방식, OAuth 방식, CORS 제약이 있는 API까지 단계적으로 경험할 수 있습니다.

## 주의할 점도 있다

다만 public-apis를 그대로 신뢰하면 안 됩니다. 무료 API는 언제든 정책이 바뀔 수 있고, 요청 제한이 낮거나, 문서가 오래됐거나, 서비스가 중단된 경우도 있습니다.

그래서 실제 제품에 쓰기 전에는 몇 가지를 확인해야 합니다. 최근에도 응답하는지, rate limit은 어느 정도인지, 상업적 사용이 가능한지, 개인정보나 민감한 데이터를 보내도 되는지, 장애 시 대체 경로가 있는지 살펴봐야 합니다.

무료 API는 실험과 프로토타입에는 훌륭하지만, 운영 서비스에서는 SLA와 데이터 품질, 비용 구조를 따로 검토해야 합니다.

## 실무자가 볼 포인트

public-apis는 단순히 “공짜 API 목록”이 아닙니다. 개발자가 빠르게 아이디어를 검증하고, 데모를 만들고, API 연동 경험을 쌓는 데 좋은 출발점입니다.

개인적으로는 새 프로젝트를 시작할 때 먼저 이 저장소를 훑어보는 방식을 추천합니다. 내가 만들려는 기능에 이미 쓸 수 있는 공개 데이터가 있는지 확인하면, 기획 범위와 구현 속도가 확 달라집니다.

좋은 API 하나는 작은 프로젝트의 완성도를 크게 올려줍니다. public-apis는 그 API를 찾기 위한 지도에 가깝습니다.
