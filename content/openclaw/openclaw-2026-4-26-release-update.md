# OpenClaw 2026.4.26 업데이트: 더 안전하고 덜 흔들리는 실행 환경으로

OpenClaw 2026.4.26은 화려한 새 기능보다, 실제로 오래 쓰는 사람에게 중요한 부분을 차분히 다듬은 안정화 업데이트에 가깝습니다.  
게이트웨이 보안, 세션 실행, 플러그인 로딩, 브라우저 제어, Google Meet 연동, 업데이트 방식까지 전반적으로 “운영 중 덜 깨지고, 예측 가능하게 동작하도록” 손본 버전입니다.

이번 업데이트의 핵심은 한마디로 **신뢰성**입니다. AI 에이전트 도구는 단순히 한 번 명령을 잘 수행하는 것보다, 여러 세션과 플러그인, 브라우저, 외부 모델 제공자를 오가면서도 안정적으로 이어지는지가 중요합니다. 그런 면에서 v2026.4.26은 기반을 꽤 꼼꼼하게 다진 릴리스입니다.

## 게이트웨이와 토큰 보안 개선

가장 눈에 띄는 변화 중 하나는 디바이스 토큰 처리 방식입니다.  
이번 버전에서는 공유 또는 관리자 `device.token.rotate` 응답에서 회전된 bearer token이 그대로 노출되지 않도록 조정되었습니다. 토큰 기반 클라이언트가 같은 디바이스 안에서 필요한 handoff를 유지할 수는 있지만, 불필요하게 민감한 값이 응답에 드러나는 상황은 줄인 것입니다.

작은 수정처럼 보일 수 있지만, 이런 변화는 운영 환경에서 꽤 중요합니다. 자동화 도구나 로그, 중간 프록시를 거치는 구조에서는 토큰이 응답에 남는 것만으로도 리스크가 생길 수 있기 때문입니다. 보안은 대개 거창한 기능보다 이런 디테일에서 차이가 납니다.

## 세션 실행과 서브에이전트 안정성

`sessions_spawn` 관련 개선도 포함되었습니다. 모델 override를 사용할 때, 설정된 bare model alias를 대상 에이전트 런타임의 기본 provider 기준으로 해석하도록 바뀌었습니다. 쉽게 말하면, 세션을 새로 띄울 때 모델 이름 해석이 더 일관적으로 동작하도록 다듬은 것입니다.

서브에이전트 쪽에서는 명시적으로 같은 에이전트를 호출하는 `sessions_spawn(agentId=...)` 요청에도 `subagents.allowAgents` 제한이 적용됩니다. 이는 권한과 허용 범위를 더 엄격히 지키기 위한 변화입니다. 또한 서브에이전트 완료 결과 전달 경로도 보강되어, 애매한 상태에서 잘못된 곳으로 결과가 흘러가는 위험을 줄였습니다.

ACP 관련해서는 명시적 부트스트랩은 실행할 수 있지만, `acp.dispatch.enabled=false` 상태에서는 자동 dispatch가 여전히 차단되도록 정리되었습니다. 자동 실행과 명시 실행의 경계를 더 분명히 한 셈입니다.

## 업데이트 과정도 더 안전하게

CLI 업데이트 방식도 개선되었습니다. npm global 업데이트를 바로 덮어쓰는 대신, 검증된 임시 prefix에 먼저 설치한 뒤 교체하는 방식으로 바뀌었습니다.  
이 변경은 업데이트 중간에 일부 파일만 섞여 설치되는 “mixed install” 문제를 예방하기 위한 것입니다.

도구 자체가 업데이트 도중 불안정해지는 경험은 사용자 입장에서 꽤 피곤합니다. 이번 수정은 그런 상황을 줄이는 실용적인 개선으로 볼 수 있습니다.

## 브라우저 제어와 Google Meet 개선

Control UI와 Talk 쪽에서는 Google Live 브라우저 세션을 WebSocket 위에 유지하고, endpoint 검증을 강화하며, relay 세션 수를 제한하는 변경이 들어갔습니다. 오래된 native voice 버튼도 제거되어 UI 혼선을 줄였습니다.

Google Meet 관련 개선도 꽤 구체적입니다. 브라우저 제어를 통해 미디어 권한을 부여하고, Chrome 오디오 기본값을 BlackHole 2ch로 고정하며, 로컬 Chrome 참가를 OpenClaw 브라우저 프로필을 통해 라우팅하도록 조정했습니다.  
즉, 회의 참여나 오디오 라우팅처럼 실제 사용 중 삐끗하기 쉬운 부분을 다듬은 업데이트입니다.

## 플러그인 발견·설치·시작 과정 정리

플러그인 시스템도 폭넓게 보강되었습니다. symlink 처리, 테스트 파일 스캔 제외, trusted peer link, 프로필 인식 destination, 중복 경고 억제, realpath 캐싱 등이 개선되었습니다.

또한 게이트웨이 플러그인 설정이 잘못된 경우 문제 있는 부분을 격리하고, malformed plugin tool은 건너뛰도록 바뀌었습니다.  
이건 꽤 반가운 변화입니다. 플러그인 하나가 이상하다고 전체 시스템이 무너지는 것보다, 문제 있는 부분을 격리하고 나머지는 계속 동작하는 편이 훨씬 운영 친화적이기 때문입니다.

Windows 환경의 플러그인 및 채널 로딩 경로 정규화도 포함되어, 플랫폼별 경로 차이에서 생기는 문제도 줄어들 것으로 보입니다.

## 모델 제공자와 로컬 모델 지원 보강

이번 릴리스는 provider 쪽도 세밀하게 손봤습니다. LiteLLM custom base URL onboarding, OpenRouter의 retired model 및 reasoning 처리, Groq와 LM Studio의 reasoning effort 값, 로컬 모델 chat completions adapter 등이 개선되었습니다.

OpenCode thinking level 지원도 추가되어, 에이전트별 사고 강도나 실행 방식과 관련된 설정이 조금 더 유연해졌습니다. 로컬 모델과 외부 provider를 함께 쓰는 환경에서는 이런 호환성 개선이 체감될 가능성이 큽니다.

## 작지만 중요한 안정성 수정들

그 외에도 broken pipe stream error를 치명적 오류로 보지 않도록 처리하고, proxy 설정에서 `ALL_PROXY`와 `all_proxy`를 존중하도록 바뀌었습니다. Docker CA 인증서 번들, MCP `type:http` 정규화, `doctor --fix` 런타임 의존성 진행 표시, Ollama WSL2 경고 중복 방지 같은 개선도 들어갔습니다.

Discord의 모델 선택기, 사용자 타깃, 세션 상속 처리도 개선되었고, 메시지 도구의 target 문법도 더 다듬어졌습니다. Reply와 link 이해 과정에서는 미디어와 링크 전처리가 안정화되었습니다.

## 정리: 이번 릴리스는 “운영 품질” 업데이트

OpenClaw 2026.4.26은 눈에 확 띄는 대형 기능보다, 매일 쓰는 환경에서 생기는 마찰을 줄이는 업데이트입니다.  
토큰 노출을 줄이고, 세션 실행 규칙을 명확히 하고, 플러그인을 더 안전하게 격리하며, 브라우저와 Meet 연동을 다듬고, 업데이트 과정까지 안전하게 만든 점이 핵심입니다.

이미 OpenClaw를 자동화 작업이나 에이전트 워크플로우에 활용하고 있다면, 이번 버전은 충분히 업데이트할 가치가 있습니다. 특히 여러 provider, 플러그인, 브라우저 제어, 서브에이전트를 함께 쓰는 사용자라면 안정성 면에서 체감할 부분이 많아 보입니다.
